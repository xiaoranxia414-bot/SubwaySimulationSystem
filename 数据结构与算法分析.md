### 1.1 性能优化（P0级）

#### 1.1.1 边密度O(n*m)计算 → O(1)增量更新
**问题**: 原实现每步遍历所有乘客计算每条边的乘客数，万人规模时每步千万次操作
```python
# 旧实现 (simulation_engine.py:278-286)
for u, v, edge_data in self.station_graph.get_graph().edges(data=True):
    edge_passengers = sum(1 for p in self.passengers ...)  # O(n*m)
```

**解决方案**: 维护 `edge_passenger_counts` 字典，乘客移动时增量更新
```python
# 新实现
self.edge_passenger_counts: dict = defaultdict(int)  # (u,v) -> count

def on_passenger_node_change(self, passenger, old_node, new_node, old_edge, new_edge):
    if old_edge: self.edge_passenger_counts[old_edge] -= 1
    if new_edge: self.edge_passenger_counts[new_edge] += 1
```

#### 1.1.2 节点容量检查O(n) → O(1)
**问题**: `_can_move_to` 每次线性扫描所有乘客
```python
# 旧实现 (passenger.py:241)
node_count = sum(1 for p in simulation.passengers if p.current_node == next_node)
```

**解决方案**: 维护 `node_passenger_counts` 字典
```python
self.node_passenger_counts: dict = defaultdict(int)
def can_enter_node(self, node_id: str) -> bool:
    return self.node_passenger_counts.get(node_id, 0) < capacity
```

### 1.2 算法优化（P1级）

#### 1.2.1 路径缓存键重构
**问题**: 使用 `sorted(passenger_profile.items())` 作为缓存键，粒度过细且有排序开销

**解决方案**: 使用分桶特征作为缓存键
```python
def _make_cache_key(self, start, end, mode, profile):
    if profile is None:
        profile_key = None
    elif isinstance(profile, dict):
        profile_key = (int(profile.get('speed', 1.0) * 3),
                       int(profile.get('patience', 0.5) * 3),
                       int(profile.get('familiarity', 0.5) * 3))
    else:
        profile_key = profile.get_profile_key()  # 使用PassengerProfile的桶方法
    return (start, end, mode, profile_key)
```

#### 1.2.2 路径重评估优化
**问题**: 每5步对所有乘客触发重评估，即使不在拥堵区域

**解决方案**:
1. 维护拥堵节点/边集合 `congested_nodes`, `congested_edges`
2. 仅对路径经过拥堵区域的乘客触发重评估
3. 引入重评估冷却机制（默认30步）

class DensitySpeedConfig:
    free_speed: float = 1.2    # 自由流速度
    max_density: float = 5.0   # 最大密度
    gamma: float = 0.3         # 衰减系数

    def get_speed_factor(self, density):

        return max(0.05, 1.0 - math.pow(ratio, self.gamma))
